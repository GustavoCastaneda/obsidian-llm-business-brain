# Filing Rules

Use these rules before creating or moving pages, folders, or MOCs. Apply the first matching rule.

1. Reuse an existing page, folder, or MOC when it already represents the concept.
2. File raw evidence and source summaries under `01-Sources/`.
3. Update durable synthesis under the relevant existing page in `02-Business/`.
4. Create a new page only for a durable concept that will be queried or updated again.
5. Do not create a new category for every source, meeting, document, or one-off topic.
6. Propose a new folder or MOC before creating it. Create it only after user approval or when an existing workflow explicitly requires it.
7. Create a new folder or MOC only when recurring content needs a stable navigation boundary.
8. Link new durable pages from an existing MOC and update `index.md` when the page is important for navigation.

When uncertain, file the source note, update an existing synthesis page, and add an open question instead of expanding the structure.
