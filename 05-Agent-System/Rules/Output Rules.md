# Output Rules

Use these rules when writing source notes, wiki pages, reports, memos, or citations.

1. Separate confirmed facts, inferences, and open questions.
2. Preserve source links, IDs, dates, and paths returned by connectors or source files.
3. Never invent URLs, citations, document links, dates, participants, or identifiers.
4. Prefer concise synthesis over copying full source content.
5. Link durable claims back to source notes whenever practical.
6. State uncertainty when evidence is incomplete, stale, or contradictory.
7. Write for operational use: clear language, specific next questions, and readable Markdown.
